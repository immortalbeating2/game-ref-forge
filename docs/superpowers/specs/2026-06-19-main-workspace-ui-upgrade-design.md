# 2026-06-19 主界面升级视觉方向

## 背景

本文件记录 RefForge / `灵感锻造台` 后续轮产品主界面升级的已确认视觉方向。

本次方向来自 3 张 UI 概念图探索。用户已确认：

- 选择第 3 张概念图作为主方向。
- 吸收第 1 张概念图的卡片密度。
- 吸收第 2 张概念图的字段秩序。

该决策用于后续主界面升级，不代表当前生产站已经实现该视觉方向。

## 主方向

后续主界面升级应以“灵感提炼工作台”为核心。

界面不只是展示 reference，而是帮助用户把参考素材转化为原创创作判断：

- 观察。
- 可复用原则。
- 避免复制。
- 转化方向。
- 参考价值。
- 可转化性。
- 生产就绪。
- 版权风险。

右侧详情区应从普通详情面板升级为更完整的结构化提炼工作区，但来源与安全信息仍要位于创意提炼之前。

## 吸收第 1 张：卡片密度

主界面应保留较高扫描密度：

- 中央 reference cards 仍是主要浏览入口。
- 卡片尺寸稳定，缩略图比例稳定。
- 卡片必须直接显示来源、分类、授权、公开状态和质量状态。
- 标签预览应紧凑，不因长标签撑开布局。
- 选中态明显，但不使用夸张发光或强装饰。

卡片密度的目标是让用户能快速横向比较多个参考，而不是把每条 reference 做成展示海报。

## 吸收第 2 张：字段秩序

主界面应吸收资料库 / 研究档案式的字段组织：

- 字段分组清晰：来源、分类、安全、评分、标签、灵感。
- 表单和详情区字段顺序一致，减少添加、编辑、复核之间的认知切换。
- 评分、标签和灵感条目要有明确结构，不只是一组自由文本框。
- 列表、筛选和详情面板中的字段命名保持一致。
- 后续可考虑在中心区提供更强的列表 / 表格视图，但不应替代卡片浏览作为默认体验。

字段秩序的目标是让 reference 数量增长后仍能维护，不把灵感提炼变成杂乱笔记。

## 视觉和交互原则

- 中文优先，保留清晰的 `中文 / EN` 切换入口。
- 首屏仍是工作台，不做营销页或 hero。
- 暗色中性底色，使用多类别小面积色彩作为识别辅助。
- 避免单一紫蓝、单一深蓝或强霓虹游戏 HUD 风格。
- 控件密度可以提高，但必须保持桌面和移动端可读。
- 右侧 `灵感提炼` 区域可以更大、更像编辑工作台。
- 安全、授权、公开状态不能被创意字段挤到次要位置。
- 删除、公开、授权相关操作要保持谨慎表达。

## 后续实现边界

后续主界面升级建议作为新的阶段型开发处理，不能直接混入小修。

建议新分支：

- `codex/round-6-workspace-ui-upgrade`

建议先写实现计划，再进入代码：

- 明确哪些只是 CSS / layout 调整。
- 明确哪些会改变组件结构。
- 明确是否新增列表视图、分组视图或详情 tabs。
- 明确移动端是否同步升级。
- 明确生产 CRUD 复测仍然是上线前阻塞项。

## 验收要点

- 默认界面体现“灵感提炼工作台”，不是素材下载站或公共图库。
- 中央 reference cards 保持高密度扫描能力。
- 右侧字段结构比当前版本更清晰，且来源与安全优先。
- 评分、标签、灵感条目在只读和编辑态中都易于理解。
- 中文首选和 English 切换保持可用。
- 桌面和移动端无明显重叠、溢出或不可点击控件。
- 生产部署前必须完成可见浏览器中的 CRUD smoke，或记录自动化阻塞并获得人工确认。
