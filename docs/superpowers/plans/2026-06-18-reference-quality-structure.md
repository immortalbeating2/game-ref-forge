# Reference Quality Structure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add multi-dimensional scores, richer tags, structured inspiration extraction, and review quality status to RefForge while preserving existing CRUD compatibility.

**Architecture:** Extend the existing single `references` table with optional/defaulted fields, then update typed reference helpers, draft conversion, localization, and the current workspace UI. Keep API routes unchanged and validate compatibility through tests plus local and production CRUD smoke.

**Tech Stack:** vinext/React, TypeScript, Drizzle ORM, D1-compatible SQLite, Vitest, Sites deployment.

---

## Scope

This plan implements the second fifth-round subtask after localization. It does not include public publishing, media upload, AI analysis, or multi-table normalization.
