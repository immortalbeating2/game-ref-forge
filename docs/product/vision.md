# Product Vision

## Identity

- Project: `game-ref-forge`
- Product: `RefForge`
- Chinese name: `灵感锻造台`
- Category: game asset reference research desk

RefForge helps collect game asset and game design references, classify them, and extract reusable design principles for original asset creation.

It is not a download site, not a file mirror, and not a copyright workaround. Its core value is turning references into structured creative thinking.

## User

The first user is an individual game creator or small-team developer who needs a private research workstation before building original game assets.

They need to:

- Save source links without losing attribution context.
- Group references by asset type and creative purpose.
- Understand why a reference is useful.
- Capture what can be transformed into original work.
- Track what must not be copied or publicly displayed.

## Goals

- Collect reference links from game asset and game design sources.
- Normalize different source formats into one reference card model.
- Classify references across game asset categories and creative use cases.
- Make every reference explain what idea, rule, or principle it contributes.
- Track source, author, license, attribution, and public-safety status.
- Keep the first version portable enough to move away from Sites/D1 later if needed.

## Non-Goals

- Do not analyze all games from the beginning of game history.
- Do not bulk crawl large asset archives.
- Do not redistribute copyrighted game assets.
- Do not host downloadable third-party asset packs.
- Do not make automatic legal judgments about reuse rights.
- Do not open public contribution or community workflows in v1.
- Do not create a separate original asset management system in v1.

## First-Version Scope

The first version is private and review-only:

- Add a reference by URL.
- Preview metadata from the URL when possible.
- Manually edit metadata before save.
- Assign normalized asset and source classifications.
- Record license and public-safety status.
- Write observation, principle, transformation, and avoid-copying notes.
- Search and filter saved references.
- Edit and delete records.

The first version should feel like a working research desk on the first screen, not a landing page.

## Product Principles

- Private by default.
- Source-first and attribution-aware.
- Ideas and principles matter more than media copying.
- Metadata extraction helps the user but never blocks manual saving.
- Classification should be useful for later creative retrieval, not just neat labeling.
- Public sharing can come later only after the safety model has proven reliable.
